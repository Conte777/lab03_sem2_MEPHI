### Задание: Реализация системы управления таблицей данных с двумя ключами

#### Описание:
Разработать программу для управления таблицей, содержащей элементы с двумя ключами. Таблица должна поддерживать добавление, поиск и удаление элементов. Для первого ключа используется прямая адресация с проверкой занятости ячеек, а для второго ключа — хеширование с использованием связанного списка для разрешения коллизий. Элементы таблицы содержат информацию, представленную динамическими строками. Программа должна обрабатывать и выводить сообщения об ошибках, таких как дублирование ключей, переполнение таблицы или некорректные данные.

#### Основные функции программы:
1. **Создание таблицы**:
   - Программа должна уметь создавать таблицу с заданными размерами для первого и второго ключевых пространств.
   - Использовать динамическое выделение памяти для структуры таблицы и элементов.

2. **Добавление элементов**:
   - Реализовать функцию добавления элемента с двумя ключами.
   - Проверка уникальности первого ключа должна осуществляться путем прямого поиска в таблице по первому ключевому пространству.
   - Для второго ключа необходимо реализовать хеширование с разрешением коллизий с помощью связанного списка.
   - В случае дублирования первого или второго ключа программа должна возвращать сообщение об ошибке.

3. **Поиск элементов**:
   - Реализовать функцию поиска элемента по первому или второму ключу.
   - Для первого ключа осуществляется прямой поиск, для второго — поиск в хеш-таблице, используя связанный список для разрешения коллизий.

4. **Удаление элементов**:
   - Реализовать функцию удаления элемента по первому или второму ключу.
   - Удаление должно учитывать возможность коллизий в хеш-таблице для второго ключа.

5. **Обработка ошибок**:
   - Программа должна обрабатывать и выводить сообщения об ошибках при:
     - Дублировании ключей.
     - Переполнении таблицы ключевого пространства.
     - Некорректных данных, вводимых пользователем.
   - Все ошибки должны быть обработаны в отдельных функциях, с выводом соответствующих сообщений.

6. **Работа с динамическими строками**:
   - Все ключи и информация для элементов таблицы должны быть представлены динамическими строками.
   - Необходимо реализовать функции для работы с динамическими строками, включая их:
     - Создание и выделение памяти.
     - Сравнение строк.
     - Копирование строк.
     - Освобождение памяти.

#### Дополнительные требования:
- Программа должна быть структурирована и разделена на модули. Основные модули:
  - Работа с таблицей (функции создания, добавления, поиска, удаления элементов).
  - Работа со строками (функции динамического управления строками).
  - Обработка ошибок (вывод сообщений об ошибках).
  
- Все операции должны корректно освобождать память, чтобы избежать утечек.

#### Пример работы программы:
1. Пользователь создает таблицу с размером первого ключевого пространства, равным 10, и второго — 15.
2. Пользователь добавляет элемент с ключами `key1 = "A1"`, `key2 = "B1"`, информация — произвольная строка.
3. Пользователь пытается добавить элемент с тем же первым ключом, и программа выводит сообщение об ошибке.
4. Пользователь выполняет поиск по первому и второму ключу, программа возвращает соответствующий элемент.
5. Пользователь удаляет элемент по первому ключу.
6. Программа выводит сообщение об успешном удалении.

Этот проект является учебной задачей по работе с хешированием, динамическим управлением памятью и обработкой ошибок в системе управления данными.
